# C端充值商城 PRD

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 海外版充值商城 C 端 |
| 文档版本 | v1.0 |
| 文档范围 | C 端商城首页、登录、角色选择、商品浏览、商品详情、限额拦截、收银台、支付结果 |
| 当前实现形态 | 纯前端 Demo + Mock API |

## 2. 产品背景

海外版充值商城面向海外服用户提供网页充值能力。用户进入商城后，可切换语言，查看对应语言下的页头、Banner、商品、充值说明和页尾信息；登录并选择角色后，才可查看可购买商品并发起支付。系统需要覆盖日本、美国注册地的未成年人充值限制，以及 PayPal、Konbini 等海外支付方式的差异化流程。

## 3. 产品目标

1. 提供海外服充值商城的完整 C 端浏览和支付闭环。
2. 支持简中、繁中、英语、日语、韩语等多语言展示。
3. 用户登录后必须先完成账号状态校验和角色选择，再进入商品列表。
4. 支持按注册地和年龄展示日本、美国充值限制，并在超限时阻断支付。
5. 支持 PayPal、Konbini 及移动端 H5 支付中断/回流确认流程。
6. 页头、页尾、Banner、充值说明等内容均可由管理后台配置并在 C 端展示。
7. 商品价格根据用户 IP 对应的 country code 自动切换；未配置地区回退默认币种和默认价格。

## 4. 用户角色

| 角色 | 说明 | 核心诉求 |
| --- | --- | --- |
| 未登录用户 | 首次进入商城或未登录状态用户 | 查看基础页面内容，发起登录 |
| 已登录未选角用户 | 登录成功但未选择角色 | 选择需要充值的游戏角色 |
| 已登录已选角用户 | 已完成角色选择 | 查看商品、发起充值、完成支付 |
| 受限用户 | 账号封禁、未解锁商店、未成年人限额用户 | 获得明确原因提示和客服入口 |

## 6. 核心业务流程

### 6.1 用户登录与角色选择流程

```mermaid
flowchart TD
  A[用户进入充值商城] --> B{是否已登录}
  B -- 否 --> C[展示登录入口和基础页面内容]
  C --> D[点击登录]
  D --> E[打开登录弹窗]
  E --> F[输入手机号/验证码或密码并勾选协议]
  F --> G{登录校验是否通过}
  G -- 否 --> H[停留登录弹窗并提示错误]
  G -- 是 --> I[关闭登录弹窗]
  I --> J[校验账号状态]
  J --> K{账号是否封禁}
  K -- 是 --> L[展示账号封禁提示弹窗]
  L --> M[用户关闭或联系客服]
  K -- 否 --> N[在登录页面展示角色选择弹窗]
  N --> O{用户是否选择有效角色}
  O -- 否 --> N
  O -- 是 --> P[保存当前角色]
  P --> Q[进入商品列表页面并加载商城数据]
```

### 6.2 切换角色流程

```mermaid
flowchart TD
  A[已登录已选角用户] --> B[点击切换角色]
  B --> C[打开角色选择弹窗]
  C --> D[默认选中当前角色]
  D --> E{用户选择角色并确认}
  E -- 未选择或选择空角色 --> C
  E -- 已选择有效角色 --> F[更新当前角色]
  F --> G[重新加载商城数据和限购状态]
```

### 6.3 商品购买流程

```mermaid
flowchart TD
  A[用户点击商品卡片/购买入口] --> B{是否登录}
  B -- 否 --> C[打开登录弹窗]
  B -- 是 --> D{账号是否封禁}
  D -- 是 --> E[展示封禁提示]
  D -- 否 --> F{角色是否已选择}
  F -- 否 --> G[打开角色选择弹窗]
  F -- 是 --> H{商店是否已解锁}
  H -- 否 --> I[展示商店未解锁提示]
  H -- 是 --> J{账号是否可充值}
  J -- 否 --> K[展示注册地/年龄限制弹窗]
  J -- 是 --> L{商品是否可购买}
  L -- 否 --> M[展示售罄/限购/超限原因]
  L -- 是 --> N[打开商品详情页]
  N --> O[用户选择数量]
  O --> P[点击立即支付]
  P --> Q{是否触发日本/美国限额}
  Q -- 是 --> R[在商品详情页后展示限额提示弹窗]
  Q -- 否 --> S[关闭商品详情并打开收银台]
```

### 6.4 PC 收银台流程

```mermaid
flowchart TD
  A[打开收银台] --> B[展示订单信息和支付方式]
  B --> C{选择支付方式}
  C -- 信用卡/借记卡、Apple Pay、Google Pay --> D[模拟即时支付]
  D --> E{支付是否成功}
  E -- 成功 --> F[展示支付成功弹窗/页面]
  E -- 失败 --> G[展示支付失败结果]
  C -- PayPal --> H[打开 PayPal 支付页面]
  H --> I{用户是否完成支付}
  I -- 完成支付 --> J[通知订单支付成功]
  J --> F
  I -- 中途返回/关闭 --> K[展示待支付确认弹窗]
  K --> L{用户选择}
  L -- 取消 --> B
  L -- 我已支付 --> M[展示查询中 toast]
  M --> N{查询订单是否已支付}
  N -- 是 --> F
  N -- 否 --> O[PC 展示未查询到支付结果 toast]
  C -- Konbini --> P[打开 Konbini 支付页面]
  P --> Q{用户返回商城}
  Q --> R[在收银台上方展示 Konbini 待支付确认弹窗]
  R --> S[点击去确认]
  S --> T[关闭收银台并回到商城页面]
```

### 6.5 移动端 H5 支付回流流程

```mermaid
flowchart TD
  A[移动端打开收银台] --> B[选择支付方式]
  B --> C[创建待支付订单]
  C --> D[跳转或唤起外部支付页/App]
  D --> E{用户是否返回商城}
  E -- 否 --> D
  E -- 是 --> F[查询订单支付状态]
  F --> G{是否已支付}
  G -- 是 --> H[展示支付成功弹窗/页面]
  G -- 否 --> I[展示待支付确认弹窗]
  I --> J{用户操作}
  J -- 取消 --> K[回到收银台继续支付]
  J -- 我已支付 --> L[展示查询中 toast]
  L --> M{再次查询是否已支付}
  M -- 是 --> H
  M -- 否 --> N[移动端展示未查询到结果弹窗]
  N --> O[点击确认关闭收银台并回到商城]
```

### 7.7 充值限额

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| C-LIMIT-01 | 注册地判断 | 按注册时 IP 判断，日本和美国适用不同规则；注册地和用户信息可联系客服修改 |

## 8. 异常与边界场景

| 场景 | 处理方式 |
| --- | --- |
| 未登录点击购买 | 打开登录弹窗 |
| 登录账号被封禁 | 展示封禁提示，隐藏角色选择，不进入商品列表 |
| 角色未解锁商店 | 展示未解锁提示，商品列表为空 |
| 无效角色 | 不允许确认选择 |
| 商品下架或不存在 | 提示商品不可用 |
| 超出限购数量 | 阻止下单并提示最多可购数量 |
| 超出日本月限额 | 阻止支付并弹出充值限制提示 |
| 美国未成年人未验证 | 阻止支付并提示需家长管控验证 |
| PayPal 中途返回 | 弹出待支付确认，取消回收银台，已支付触发查询 |
| Konbini 中途返回 | 弹出 Konbini 专属待支付确认，确认后关闭收银台 |
| 支付结果未查询到 | PC 展示 toast，移动端展示弹窗并关闭收银台 |

## 9. 数据与接口

| 接口能力 | 用途 |

| 页面配置 | `languages`、`header`、`footer`、`banners` | 控制多语言内容展示 |

## 10. 非功能需求

1. PC 和移动端 H5 均需适配，重点保证收银台和弹窗在小屏可用。
2. 支付流程必须保留状态，避免用户从外部支付页返回后丢失订单。
3. 多语言缺失时需有明确兜底，避免页面出现空白关键文案。
4. 限额和封禁提示必须在关键路径前置，避免用户进入不可完成的支付流程。
5. C 端展示依赖已发布配置，草稿内容不应直接影响用户页面。
